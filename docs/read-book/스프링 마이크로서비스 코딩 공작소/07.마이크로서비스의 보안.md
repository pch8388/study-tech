# OAuth2 소개
- 보호 자원(resource server) : 보호하려는 자원, 인증된 사용자만 액세스
- 자원 소유자(resource owner) : 서비스 호출자 (엔드 유저)
- 애플리케이션(client) : 사용자를 대신해 서비스를 호출할 애플리케이션
- 권한 부여 서버(authorization server) : 자원 소유자를 인증하고 자원 소유자의 승인을 받아서 클라이언트에게 토큰을 발급
